## Summary

Revia now defaults to **dark mode**, assigns every user a unique **username** (customizable in Settings), shows usernames on public decks in Explore, and supports **sign-in with username** as well as email.

---

## Added

- **Usernames** — Auto-assigned on first login (e.g. `swift-raven-1926`); change anytime in **Settings → Username**
- **Username on public decks** — Explore shows `@username` beside each public deck title
- **Sign in with username** — Login accepts email or username; usernames resolve to email server-side
- **Account API** — `GET/PATCH/POST /api/account` for profile and username updates; `GET /api/auth/resolve-email` for login lookup

## Changed

- **Default theme** — Dark mode for new users; light mode kept if explicitly chosen before
- **Login form** — Field label "Email or username" on sign-in
- **Account settings** — Shows `@username` alongside email

## Database

- `users.username` column added (`String?`, unique)
- Backfill script: `npx tsx scripts/backfill-usernames.ts` assigns usernames to existing users

## Migration notes

Before or immediately after deploy:

```bash
cd revia
npx prisma db push --accept-data-loss
npx tsx scripts/backfill-usernames.ts
```

New sign-ups receive a random username automatically via account sync.
