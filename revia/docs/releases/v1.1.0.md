## Summary

Users can publish decks as public or keep them private. Public decks appear in **Explore** for all learners. Settings now shows the app version at the bottom of the page.

---

## Added

- **Deck visibility** — Toggle each deck between **Private** (default) and **Public** from the deck detail page
- **Explore page** (`/explore`) — Replaces Search in navigation; personal library search at the top, public deck discovery below
- **Public deck browsing** — Non-owners can view public decks, browse lessons, and study cards in read-only mode (no review ratings saved)
- **Explore API** — `GET /api/explore` lists searchable public decks with author name and card count
- **App version** — Settings footer displays `Revia v1.1.0` from `package.json`

## Changed

- Nav label **Search** → **Explore** (Compass icon); `/search` redirects to `/explore`
- Deck detail API returns `isOwner` and supports read access for public decks
- Lesson and card APIs allow read access on public decks; mutations remain owner-only
- Study viewer supports read-only mode (Next card instead of rating buttons)

## Database

- `decks.is_public` column added (`Boolean`, default `false`), indexed for public deck queries

## Migration notes

Run `npx prisma db push` (or apply migration) before deploy. Existing decks remain private until the owner publishes them.
