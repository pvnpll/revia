## Summary

Settings is now a clean hub with separate pages for each option. Signup and email confirmation are more reliable, with clearer errors for rate limits and misconfigured redirect URLs.

---

## Added

- **Settings hub** — Main Settings page lists options only; each opens its own page
  - Account, Username, Appearance, Import deck, Create deck, About
- **Auth error helpers** — Friendly messages for email rate limits, unconfirmed email, and duplicate accounts
- **`NEXT_PUBLIC_APP_URL`** — Canonical URL for confirmation email redirects
- **`npm run supabase:fix-production-url`** — Script to fix Supabase Site URL and optional auto-confirm

## Changed

- **Signup flow** — No longer calls sign-in immediately after sign-up (avoids Supabase rate limits)
- **Settings layout** — Back navigation to Settings from each sub-page

## Fixed

- Confirmation emails pointing at wrong domain when Site URL is misconfigured
- “Security purposes / 36 seconds” error from double auth requests on signup

## Migration notes

1. Set **Vercel env:** `NEXT_PUBLIC_APP_URL=https://revialearn.vercel.app`
2. Fix **Supabase Site URL** (must include the dot: `revialearn.vercel.app`)
3. Optional: run `npm run supabase:fix-production-url` to enable auto-confirm and skip confirmation emails on free tier
