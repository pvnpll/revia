## Summary

Visitors can use Revia **without signing in**: browse **Explore**, open **public decks**, flip through cards with **swipe/tap navigation**, and practice lessons. **Daily Review**, personal library, import, and settings still require an account.

---

## Added

- **Guest access** — Land on Explore when not signed in; optional auth routes for public deck reads
- **`getOptionalUserId()`** — Read-only APIs (explore, deck, lessons, cards, practice with `deckId`) work without a session
- **`LoginRequired`** — Sign-in prompts for library, Daily Review, import, and personal practice
- **`useAuthSession`** — Client auth state for shell, explore, and practice flows
- **Guest practice** — Swipe right / tap right edge for next card; swipe left / tap left for previous; tap center to reveal (no 1–5 ratings, no saved progress)
- **App version on Explore** — `Revia vX.Y.Z` at the bottom of the page (guest and signed-in)

## Changed

- **Home redirect** — Guests → `/explore`; signed-in users → `/practice`
- **Middleware** — Guest routes: `/explore`, `/practice`, `/decks/[id]`
- **Shell** — Guest header shows Sign in / Sign up; nav shows Explore + Practice only
- **Explore** — Guests see public decks first; library search hidden until signed in
- **Lessons on public decks** — Full card browse for guests via `/practice?deckId=…&lessonId=…`
- **Practice vs review UI** — Practice mode uses Question/Answer labels; review keeps rating scale

## Fixed

- **Guest practice reliability** — Portal overlay renders immediately; lesson links use direct navigation
- **Practice nav prefetch** — No unauthorized `/api/practice/cards` prefetch for guests
